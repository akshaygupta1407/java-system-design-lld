# Module 7: LLD Interview Practice

This module is for practicing full LLD problems like an interview.

You will solve each problem yourself first. Then I will review your design, point out gaps, and help you improve it.

---

## How This Module Works

For each question:
1. Read the problem statement.
2. Ask clarifying questions.
3. Write requirements and assumptions.
4. Identify core entities.
5. Define class responsibilities.
6. Write code or class skeletons.
7. Send your solution for review.

I will judge your answer on:
1. Requirement clarity
2. Correct entity modeling
3. SOLID usage
4. Extensibility
5. Simplicity
6. Code readability
7. Edge cases

---

## Interview Answer Format

Use this format when solving:

```text
1. Clarifying Questions
2. Assumptions
3. Core Entities
4. Class Responsibilities
5. Design Patterns Used
6. Edge Cases
7. Code
8. Possible Improvements
```

## Review Rubric

When you submit a solution, I will review it using this rubric:

| Area | What I Check |
| --- | --- |
| Requirements | Did you clarify the right scope? |
| Entities | Are the main objects modeled correctly? |
| Responsibilities | Does each class have one clear job? |
| Relationships | Are associations and dependencies clean? |
| Extensibility | Can the design handle future changes? |
| Patterns | Are patterns used only where useful? |
| Edge Cases | Are failure cases handled? |
| Code Quality | Is the solution readable and maintainable? |
